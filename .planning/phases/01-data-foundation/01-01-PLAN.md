In src/utils/parsers.js, line 38, expand the iLocation column regex to also match a CSV column named "Zone" (case-insensitive, anchored to exact match):

```javascript
// BEFORE (line 38):
const iLocation = col(/general\s*location|activity.*location/i);

// AFTER:
const iLocation = col(/general\s*location|activity.*location|^zone$/i);
```

This is a defensive expansion. If the uploaded metadata CSV uses "Zone" as the column header instead of "General Location" or "Activity Location", the existing regex silently returns -1, causing all activities to have `location: ""`. Adding `^zone$` (anchored to avoid matching unrelated columns like "Zone Notes") handles this case with zero risk to existing behavior.

No other changes to parsers.js.
